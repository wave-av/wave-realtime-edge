## [Unreleased]

### Added
- feat(media-tap): E-MEDIA-TAP (#74) — the ONE room-subscribe surface so egress and #85 perception don't each open a parallel SFU subscription (transform-once fan-out-many). New `src/media-tap.ts` (pure engine: `MediaTap.subscribe(consumerId, selector) → handle` + `publish(frame)` fan-out; a bounded per-consumer queue applies backpressure via drop-oldest; `publish` runs no consumer code so one slow/absent consumer never stalls the source or its peers; `selectorMatches` filters by kind/track/participant; `stats()` receipt) + the P3 consumer contract (`MediaConsumer` + `pumpConsumer`, which isolates a throwing `onFrame`). The `RoomDO` frame sink (`feedRecorderFrame`, also the `recorder-frame` intent) publishes each decoded frame into the room's `mediaTap`, resolving `(participant,kind)` from the registry SSOT — behind `MEDIA_TAP_ENABLED` (default OFF → tap built but publishes nothing, prod byte-identical). Egress-router (#75) and #85 perception (#76) attach through this sole API. Prod arm = a Jake-named crossing.
- feat(presence): E-ROOMS P4 (#73) — client presence / state-sync / data channel over a hibernatable WebSocket the `RoomDO` owns. New `src/presence.ts` (pure engine: `projectRoomView` role-aware client-safe projection, `parseInbound` hardening, `PresenceHub` over an injected `SocketRegistry`; + thin DO glue). `RoomDO` gains a `presence` intent (WS upgrade → welcome snapshot), a `webSocketMessage` hibernation handler, and an authoritative `state` broadcast (monotonic `version` = conflict-free ordering) after each mutating intent (join/publish/leave). Data-channel messages fan out to the OTHER participants; the waiting list is projected to hosts only; SFU sessionIds / ban list / admitted markers are never leaked. Route `GET /v1/realtime/rooms/:room/presence` behind `PRESENCE_ENABLED` (default OFF → inert, 501 fall-through, prod behavior byte-identical). Replaces the poll-for-state gap so clients can subscribe. Prod arm = a Jake-named crossing.
- feat(rt-encoder): per-leg capability-negotiation layer (#86, inert/verify-only). Full `CapabilityDescriptor` (`server/descriptor.mjs`) — adds a `ffmpeg -decoders` probe (`server/decode.mjs` `parseDecoders`/`decodableCodecs`), a static env-driven `transports` list + region helper (`server/transports.mjs`), and `maxResFps` — composed into the additive `/capabilities` response (existing `{hwaccels, codecs}` encode output byte-stable). The pure per-leg selector `selectLeg` (`server/leg-select.mjs`) negotiates encode-codec ⟂ transport ⟂ decode-codec with honest-negative typed exclusions (`CODEC_UNAVAILABLE`/`DST_DECODE_UNSUPPORTED`/`NO_COMMON_TRANSPORT`/`TRANSPORT_NOT_ACTIVATED`/`REGION_PLACEMENT_VIOLATION`). No live behavior change; wires into the coordinator/gateway resolver in a follow-up (C1/G1).
- ci(deploy): deploy pipeline (`.github/workflows/deploy.yml`, workflow_dispatch) — install · typecheck · test · `wrangler deploy` (#30).
- wrangler: `ROOM` Durable Object binding + `v1` migration (`new_sqlite_classes: ["RoomDO"]`), `[observability]`, `workers_dev = false`, `GATEWAY_BASE_URL` var, and documented secrets (`CF_CALLS_APP_ID/SECRET`, `WAVE_SERVICE_TOKEN`).
- `RoomDO` re-exported from `src/worker.ts` so the DO binding/migration resolve on first deploy.
- `deploy:dry` npm script (`wrangler deploy --dry-run`).
