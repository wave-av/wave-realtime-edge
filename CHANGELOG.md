### Added
- feat(ingress-container-bridge): E-INGRESS P4 (#77) — the container-bridge ingest BACKEND, the most-owned-compute tier (`containerBridge`) and the THIRD/LAST backend behind the P1 decision core, completing the ingress backend set. New `src/ingress-container-bridge.ts`: `ContainerBridgeIngestBackend` is a router-facing adapter over the EXISTING Plane-2 control plane (`ingest-bridge.ts`, F #55 — `/start`+`/stop`, gateway-trust, server-side org, the every-15-min cron reconcile, fail-closed `<PROTO>_BRIDGE_NOT_ACTIVATED` 501s), NOT a re-mint of it. `admit(job, {org, inbound?})` routes a typed `IngestJob` through the authoritative `ingressRoute` (P1) and, ONLY when the verdict is `containerBridge`, asks an injected `ContainerBridgeClient` to start the per-protocol container leg (rist/moq — the CF-Stream-can't-carry backstop — plus the self-hosted rtmp/srt path); the push protocol the container terminates is taken from the router's decision (authoritative), never re-derived from the wire. It owns exactly ONE plane: a verdict for `cfCallsSfu` (P3) or `cfStreamLive` (P2) returns a typed `deferred` outcome, never a stray container start. **THE SSRF GUARD** — the security value the router explicitly deferred to "the backend in P4": the container is the one ingest path that can DIAL OUT (a self-hosted leg's `inbound.host` is the remote it connects to, parsed from the untrusted `/start` body), so before that host reaches the container-dial sink it is guarded (`guardIngestHost`) against internal/loopback/link-local/cloud-metadata(169.254.169.254)/CGNAT-tailnet(100.64/10) targets — the host is first CANONICALIZED through the WHATWG host parser (the same normalization a URL-based dialer uses) so an alternate IPv4 notation (decimal `2130706433`, hex `0x7f000001`, octal `0177.0.0.1`, shorthand `127.1`) can't evade the check by a spelling the OS still resolves to an internal address; then IPv4 literals are range-checked, and IPv6 literals PARSED to their numeric groups (`parseIpv6` + `isBlockedIpv6`, incl. `::` expansion + `::ffff:`/embedded-IPv4 folding) so that no textual form — fully-expanded, zero-padded, uppercase, or zone-id'd — can bypass the range check (e.g. `0:0:0:0:0:0:0:1` is blocked, not just `::1`); known-internal names rejected. A customer must never make our owned compute touch an internal address (ssrf-guard-before-user-supplied-url-fetch). `guardPullUrl` is the same guard as a reusable URL wrapper the `cfStreamLive` `urlPull` feed uses at ITS arm (one authoritative guard, not two); the residual DNS-rebinding gap is closed at the ARM slice by re-guarding the resolved IP. Health/backpressure are SURFACED, not fabricated — a container 503/at-capacity comes back as a non-2xx `ContainerBridgeResult` and is never mistaken for a live leg (durable start-failure recovery already lives in the control plane's cron reconcile). PURE + injected seam — routing/request-building/SSRF-guarding are deterministic; the only I/O (the container `/start` control fetch) is behind the `ContainerBridgeClient` interface, so the backend is unit-tested with a fake client and NO container/CF wire code ships in this slice (the concrete adapter wraps `liveIngestBridgeDeps().dispatchStart`). INERT: shares `INGRESS_ROUTER_ENABLED` with P2/P3 (one flag arms the router + all its backends), default OFF ([vars]); nothing instantiates the backend and no ingest-bridge.ts/room.ts edit is needed until the ◆ Jake-named arm crossing. 24 hermetic tests. **All three ingress backends (cfCallsSfu / cfStreamLive / containerBridge) now exist behind the P1 router — the ingress backend set is complete.**
- feat(ingress-router): E-INGRESS P1 (#77) — the inbound-ingest decision core. Ingress, like egress, is a decision function, not one target: `LiveKitIngressService` took an ingress mode (RTMP_INPUT / WHIP_INPUT / URL_INPUT) and set up the right inbound path. wave-native already has the backends — three orthogonal ingest surfaces (`whip.ts` native WebRTC → SFU, `stream-bridge.ts` CF Stream Live managed, `ingest-bridge.ts` Plane-2 per-protocol container) — but no unifying router. New `src/ingress-router.ts`: a pure, deterministic `ingressRoute(job) → {ok, backend, costRank, pushProtocol, requiresSsrfGuard} | {ok:false, reason}` over a typed `IngestJob` (source kind + target room + optional pull URL), driven by the authoritative cost-ascending `INGEST_ROUTING_TABLE` — `cfCallsSfu` (native WHIP) → `cfStreamLive` (free managed RTMP/SRT push + URL pull) → `containerBridge` (the Plane-2 backstop for RIST/MoQ that CF Stream can't carry). "Cheapest capable first" falls out of the table walk (grounded in the transport SSOT), so RTMP/SRT push takes the free managed path before the owned container. `costRank` is an honest ordinal (native WebRTC < managed CF Stream < self-hosted container-transcode), NOT a $/hour figure. An optional `maxCostRank` ceiling caps escalation (a RIST push capped at the managed rank is rejected with a reason, never silently sent to the container). Reuses the existing `IngestProtocol` SSOT via `pushProtocolOf` (no parallel enum); `mapLiveKitIngress` maps the three legacy LiveKit modes onto wave-native source kinds for E-DECOMMISSION; a `urlPull` decision carries `requiresSsrfGuard: true` so the P4 backend guards before fetch. Reads no clock/env/network and owns no media. The backends (P2–P4) call this router; nothing wires it into the worker yet (additive, inert). 19 hermetic tests.
- feat(ingress-whip-sfu): E-INGRESS P3 (#77) — the WHIP/WebRTC ingest BACKEND, the most-direct tier (`cfCallsSfu`) and second backend behind the P1 decision core. New `src/ingress-whip-sfu.ts`: `WhipSfuIngestBackend` is a router-facing negotiator that serves the `cfCallsSfu` verdict — native WebRTC WHIP ingest straight into the SFU (the existing `whip.ts` / `handleWhip` surface), no transcode, no managed-ingest hop, no container. `admit(job, {org, sdpOffer})` routes a typed `IngestJob` (`sourceKind:"whip"`) through the authoritative `ingressRoute` (P1) and, ONLY when the verdict is `cfCallsSfu`, asks an injected `WhipSfuClient` to negotiate the WHIP publish into the SFU (SDP offer → SDP answer + a WHIP resource id for trickle/teardown). It owns exactly ONE plane: a verdict for `cfStreamLive` (P2) or `containerBridge` (P4) returns a typed `deferred` outcome, never a stray negotiation. NO SSRF surface — WHIP is a push (the router sets `requiresSsrfGuard` only for `urlPull`), so the guard here is the SDP offer + org: an authless/empty org or a non-SDP offer body (`isValidSdpOffer` requires a `v=0` first line) is refused (`unroutable`) before the SFU seam, never negotiated on garbage. PURE + injected seam — routing/request-building/offer-validation are deterministic; the only I/O (the CF Calls SDP exchange) is behind the `WhipSfuClient` interface, so the backend is unit-tested with a fake client and NO SFU/CF-Calls wire code ships in this slice (the concrete adapter, reconciled against `handleWhip`, is the ARM slice). INERT: shares `INGRESS_ROUTER_ENABLED` with P2 (one flag arms the router + its backends), default OFF ([vars]); nothing instantiates the backend and no whip.ts/room.ts edit is needed until the ◆ Jake-named arm crossing. Reuses the CF Calls app pair (`CF_CALLS_APP_ID` + `CF_CALLS_APP_SECRET`) the whip.ts surface already binds. 12 hermetic tests.

New `src/ingress-cf-stream-live.ts`: `CfStreamLiveIngestBackend` is a one-shot PROVISIONER (ingress is not a tap consumer) — `provision(job, {org})` routes a typed `IngestJob` through the authoritative `ingressRoute` (P1) and, ONLY when the verdict is `cfStreamLive`, asks an injected `CfStreamLiveClient` to create a CF Stream Live input (free managed RTMP/SRT push + URL pull) for the caller's room + org. - feat(egress-cf-stream-passthrough): E-EGRESS-ROUTER P4 (#75) — the CF Stream passthrough egress BACKEND, the cheapest tier (`cfStream`) and the last of the three egress backends behind the P1 decision core. New `src/egress-cf-stream-passthrough.ts`: `CfStreamPassthroughEgressBackend` serves the `cfStream` verdict — PASSTHROUGH: record or RTMP-simulcast a track AS-IS via Cloudflare Stream, no compositing/encode. Because passthrough consumes no frames (it hands the SFU track to CF Stream, it does not composite pixels), this backend is a one-shot PROVISIONER — NOT a `MediaConsumer` like the two composite backends (P2 wave-render, P3 RunPod). `provision(job, target)` routes a passthrough `EgressJob` (`needsCompositing:false`) through the authoritative `egressRoute` (P1) and, ONLY when the verdict is `cfStream`, asks an injected `CfStreamEgressClient` to create the CF Stream output (a recording, or a simulcast live-output). It owns exactly ONE tier: a verdict for `waveRender` (P2) or `runpodNvenc` (P3) — both compositing jobs — returns a typed `deferred` outcome, never a stray provision. A simulcast pushes to an external RTMP target the caller supplies, so its destination is scheme-validated to `rtmp:`/`rtmps:` (`isValidRtmpDestination`) before it reaches the origin seam (validate-untrusted-input-before-sink) — a missing/non-RTMP destination is refused (`unroutable`), never handed to CF Stream; a `record` output needs no destination. PURE + injected seam — routing/job-building/destination-validation are deterministic; the only I/O is behind the `CfStreamEgressClient` interface, so the backend is unit-tested with a fake client and NO CF-Stream-API/wire code ships in this slice (the concrete adapter is the ARM slice). INERT: shares `EGRESS_ROUTER_ENABLED` with P2/P3 (one flag arms the router + all its backends), default OFF ([vars]); nothing instantiates the backend and no room.ts edit is needed until the ◆ Jake-named arm crossing. - feat(egress-runpod-nvenc): E-EGRESS-ROUTER P3 (#75) — the RunPod NVENC egress BACKEND, the GPU escalation backstop behind the P1 decision core. NO $/output-minute or list price is asserted — that needs a GPU-seconds-per-encoded-minute throughput bench under load (mirroring the LLM-COGS "never list before benching" rule; that bench + the concrete RunPod adapter is the ARM slice). PURE + injected seam — routing/job-building/frame-tracking/COGS arithmetic are deterministic; the only I/O is behind the `RunpodNvencClient` interface, so the backend is unit-tested with a fake client and NO endpoint/wire code ships in this slice. - feat(egress-router): E-EGRESS-ROUTER P1 (#75) — the routed-egress decision core. Egress is a decision function, not one target: LiveKit Egress did four jobs (composite recording / single-track egress / RTMP simulcast / web-composite = wave-render), so wave-native egress is a router that sends each job to the cheapest capable tier — `cfStream` passthrough (no compositing) → `waveRender` dogfood composite (#61, within a moderate envelope) → `runpodNvenc` GPU backstop (heavy multi-source / high-res / HEVC-AV1 / real-time-at-scale). New `src/egress-router.ts`: a pure, deterministic `egressRoute(job) → {ok, backend, costRank} | {ok:false, reason}` over the typed `EgressJob` shape, driven by the authoritative cost-ascending `EGRESS_ROUTING_TABLE` (config, not scattered conditionals) so "cheapest capable first" falls out of the table walk; wave-render's envelope is centralized as typed `WAVE_RENDER_CAPS`; an optional `maxCostRank` ceiling caps escalation (a valid job over its ceiling is rejected with a reason, never silently escalated). `costRank` is an honest ordinal ordering (passthrough < own-infra composite < GPU), NOT a fabricated $/hour — real GPU COGS lands with the RunPod backend (P3). Reads no clock/env/network and owns no media. The backends (P2–P4) attach as `MediaConsumer`s off the one `MediaTap` (#74) and call this router; nothing wires it into the worker yet (additive, inert). 19 hermetic tests.
